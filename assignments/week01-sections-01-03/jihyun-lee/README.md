## Redis란?

Redis는 데이터를 메모리에 저장하는 Key-Value 기반 NoSQL 데이터베이스이다.

일반적인 MySQL 같은 DB는 데이터를 주로 디스크에 저장하지만, Redis는 데이터를 주로 RAM, 즉 메모리에 저장한다. 그래서 읽기와 쓰기 속도가 매우 빠르다.

예를 들면 Redis에는 데이터를 이런 식으로 저장할 수 있다.

```
key: user:1:name
value: "지현"
```

즉, Redis는 데이터를 복잡한 테이블 구조보다는 key로 빠르게 찾아오는 구조로 사용한다.

## Redis의 장점

Redis의 가장 큰 장점은 **속도**이다.

Redis는 데이터를 메모리에 저장하기 때문에 디스크 기반 DB보다 훨씬 빠르게 데이터를 읽고 쓸 수 있다. 그래서 자주 조회되는 데이터나 빠르게 처리해야 하는 데이터를 저장할 때 많이 사용한다.

예를 들어, 사용자가 요청할 때마다 매번 DB에서 조회하면 트래픽이 많아질수록 DB에 부담이 커진다.

```
사용자 요청 → 서버 → DB 조회 → 응답
```

그런데 Redis 캐시를 사용하면 자주 조회되는 데이터를 Redis에 저장해두고, 다음 요청부터는 DB 대신 Redis에서 빠르게 가져올 수 있다.

```
사용자 요청 → 서버 → Redis 조회 → 응답
```

이렇게 하면 DB 부하를 줄이고 응답 속도를 높일 수 있다.

## Redis 주요 사용 사례

- 캐싱
    
    Redis에서 가장 많이 사용되는 방식이다.
    
    이때 Redis에 저장된 데이터는 보통 영구 저장용이라기보다는 빠른 조회를 위한 임시 저장소로 많이 사용된다.
    
- 세션 관리
    
    로그인 상태를 관리할 때도 Redis를 사용할 수 있다. 
    
    예를 들어 사용자가 로그인하면 서버는 사용자의 로그인 정보를 세션으로 저장한다. 이 세션 정보를 Redis에 저장하면 서버가 여러 대일 때도 같은 세션 정보를 공유할 수 있다.
    
- 실시간 분석 및 통계
- 메시지 큐
- 지리공간 인덱싱
- 속도 제한
- 실시간 채팅 및 메시징

## 대용량 트래픽 처리 경험 / Redis 사용 경험

대용량 트래픽을 처리하기 위해서 필수적으로 사용되는 기능이 Redis의 캐싱 기능이다.

대용량 트래픽에서는 많은 사용자가 동시에 서버에 요청을 보내는데, 사용자가 몰리면 이런 문제가 생길 수 있다.

```
사용자 10,000명 → 서버 → DB에 조회 요청 10,000번
```

이렇게 되면 DB에 부하가 커지고 응답 속도가 느려질 수 있다.

Redis 캐시를 사용하면 자주 조회되는 데이터를 Redis에 저장해두고, DB 조회 횟수를 줄일 수 있다.

그래서 Redis는 대용량 트래픽 환경에서 DB 부하를 줄이고, 응답 속도를 빠르게 하기 위해 많이 사용된다.

## Redis 기본 명령어 익히기

데이터(Key, Value) 저장하기: set

```bash
set [key 이름] [value]
```

데이터 조회하기 (Key로 Value 값 조회하기): get

```bash
get [key 이름]
```

저장된 모든 key 조회하기

```bash
keys *
```

데이터 삭제하기 (Key로 데이터 삭제하기): del

```bash
del [key 이름]
```

데이터 저장 시 만료시간(TTL) 정하기

```bash
set [key 이름] [value] ex [만료 시간(초)]
```

만료시간(TTL) 확인하기

```bash
ttl [key 이름]
```

- 키가 없는 경우 -2 반환
- 키는 존재하지만 만료 시간이 설정돼 있지 않은 경우 -1 반환

모든 데이터 삭제하기

```bash
flushall
```

## Redis에서 Key 네이밍 컨벤션 익히기

콜론(`:`)을 활용해 계층적 의미를 구분해서 사용한다.

→ 가독성: 데이터의 의미와 용도를 쉽게 파악할 수 있다.

→ 일관성: 컨벤션을 따름으로써 코드의 일관성이 높아지고 유지보수가 쉬워진다.

→ 검색 및 필터링 용이성: 패턴 매칭을 사용해 특정 유형의 Key를 쉽게 찾을 수 있다.

→ 확장성: 서로 다른 Key와 이름이 겹쳐 충돌할 일이 적어진다.
